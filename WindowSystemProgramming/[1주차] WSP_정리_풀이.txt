1. 컴퓨터 시스템의 주요 구성요소 4가지를 CPU에서 가까운 순서로 적고 
   CPU로 부터 가까울수록/멀어질수록 각 요소에 대한 장단점을 적으시오

) CPU - 캐시 - 메인메모리 - 하드디스크
cpu로 부터 가까울 수록 처리 속도가 빠르고 용량이 작다. 멀어질수록 처리 속도가 느린 대신 용량이 크다.

2. 쿨럭 신호란 무엇인가? 왜 필요한가?
) 쿨럭 신호는 동기화를 위해 필요. 쿨럭 신호가 없다면 예를 들어 출력 장치와 데이터 전송 속도의 차이로 인해
데이터가 누락되거나 하는 문제가 발생할 수 있다. 

3. 프로그램의 실행 과정을 상세하게 서술하시오
) 프로그램 실행 시(exe 파일 실행) 메인메모리에 올라감 -> I/O 버스를 통해 CPU로 이동 -> 컨트롤 유닛으로 전달, 해석 ->
ALU 에서 해석된 내용 연산 + 결과값을 중간중간 레지스터에 저장

4. 메인 메모리 특징이나 역할에 대해 아는대로 서술하시오.
) 메인 메모리는 RAM이라고 불리며 실행된 프로그램들의 데이터가 들어오게 되며
이를 CPU로 전달한다., 휘발성 저장소이므로 컴퓨터가 종료되면 내용이 날아간다는 특징이 있다.

5. 버스 인터페이스의 간단하게 역할을 서술하시오
) 버스 인터페이스는 CPU 내부에 위치하며 메인메모리로 부터 I/O버스를 통해 전달받은 데이터를 호환될 수 있도록 간단하게 해석하는 역할

6. 32bit 시스템과 64bit 시스템의 차이점 3가지 서술하시오
) 1.주소값의 크기(8바이트/16바이트)가 다르다.  2. 송/수신 가능한 데이터의 크기가 64bit 시스템이 2배 더 크다
3. 한번에 처리 가능한 데이터의 크기가 64bit 시스템이 2배 더 크다

7. 32bit 시스템을 64bit 시스템으로 포팅해야한다면 주의해야 하는 점이 무엇인가?
) 포인터형식의 데이터를 형변환할 때의 포인터의 크기값이 64bit 시스템과 다르기 때문에 이로 인한 문제가 발생 가능

8. 레지스터의 역할에 대해 명령어와 연관지어 간단하게 서술하시오
) 레지스터는 임시 저장소와 같은 개념으로 LOAD를 통해 메인 메모리의 데이터 수신, STORE로 메인 메모리에 데이터 전달을 한다.

9. 메모리에 접근하는 방식은 크게 Direct 모드와 InDirect 모드 2가지가 있다.
) Direct 모드는 메모리에 있는 데이터 값으로 접근하는 방식으로 표현할 수 있는 비트수의 한계 때문에
 모든 메모리 데이터에 접근할 수가 없고, InDirect 모드는 주소값으로 접근하는 방식으로 InDirect모드가 Direct모드의 단점을 보완할 수 있다.
